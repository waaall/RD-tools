# RD-tools 基于 QFluentWidgets 的 UI 整改计划

这是一个旧文档，不代表现在设计实现。

## 摘要
- 目标是把当前 `PySide6` 手写界面升级为统一、现代、可扩展的工具型桌面 UI，同时保持现有业务逻辑、线程模型、配置格式和模块注册方式基本不变。
- 本次整改采用 `PySide6-Fluent-Widgets==1.11.1` 的公开版基础组件，不使用 Pro 组件，不引入 `[full]` 依赖作为默认方案。
- 主体策略是“重做表现层，不重写业务层”：窗口壳、导航、页面骨架、控件体系、主题和状态反馈全部统一到 QFluentWidgets；`BatchFilesBinding`、各模块处理类、`settings.json` 和 `AppSettings` 继续沿用。
- 默认交付结果是一个以 `FluentWindow` 为壳的三页应用：`任务中心`、`设置`、`帮助`，默认页为 `任务中心`，支持现有 `Light/Dark` 主题配置。

## 关键实现变更
### 1. 应用壳与主题层
- 将主窗口从当前 `QMainWindow + QDockWidget + QStackedWidget` 重构为 `FluentWindow`，彻底移除旧 `DockPage` 方案。
- 导航项固定为三项：`任务中心`、`设置`、`帮助`。`帮助` 放到底部导航位，其余放顶部。
- 主题入口集中到单一 UI 主题模块，负责：
  - 根据现有设置 `theme` 映射到 `Theme.LIGHT` 或 `Theme.DARK`
  - 统一设置主题色 `#2F6FED`
  - 加载应用级自定义 QSS
  - 禁止页面组件再内联写零散 `setStyleSheet()`，除了极少数运行时状态色场景
- 视觉 token 固定如下：
  - 主强调色：`#2F6FED`
  - 成功色：`#2AA676`
  - 警告色：`#D49C1D`
  - 错误色：`#C55252`
  - 暗色主题主背景：`#202124`
  - 暗色主题面板：`#26282C`
  - 暗色主题卡片：`#2C2F33`
  - 文本主色：`#F5F7FA`
  - 次级文本：`#B8C0CC`
- 布局规则固定如下：
  - 页面外边距 `24`
  - 卡片内边距 `16`
  - 区块间距 `16`
  - 行内控件间距 `12`
  - 控件高度统一为 `36`
  - 卡片圆角 `10`
- 不做自定义无边框标题栏和平台特化美化；使用库默认窗口样式，避免首版引入兼容风险。

### 2. 任务中心重构
- 当前“纵向任务卡片堆叠”重构为“左任务列表 + 右任务工作区”的稳定骨架。
- 页面结构固定为三层：
  - 页头：页面标题、说明文案、当前工作目录摘要
  - 主体左栏：任务列表
  - 主体右栏：当前任务工作区
- 工作目录是全局状态，不再在每个任务卡片里重复出现。全局上下文只保留一份，所有任务共享。
- 右侧工作区固定分为两部分：
  - 顶部任务概览区：任务标题、说明、执行状态、主按钮 `执行任务`、次按钮 `清空日志`
  - 底部内容区：用 `SegmentedWidget` 切换 `处理范围` 与 `运行日志`
- `处理范围` 页签固定包含：
  - 只读路径展示框
  - `选择工作目录` 按钮
  - `全选`、`清空选择` 按钮
  - 目录数量摘要
  - 单列可勾选目录列表
- `运行日志` 页签固定包含：
  - 当前任务独立日志视图
  - 自动滚动到底部
  - 空日志时展示占位文本
- 任务列表由统一 `TaskDescriptor` 驱动，替代现在的元组列表。字段固定为：
  - `key`
  - `title`
  - `description`
  - `icon`
  - `operation_cls`
  - `default_params`
- 当前 `register_batch_operation()` 保留职责，但改为消费 `TaskDescriptor`，并把描述信息供任务列表与详情区共用。
- 运行态规则固定如下：
  - 某任务运行时，仅该任务按钮禁用
  - 任务列表项显示运行中状态
  - 详情区显示 `IndeterminateProgressRing`
  - 日志继续通过现有信号链路追加
- 反馈规则固定如下：
  - 缺少工作目录、未勾选目录、执行异常等阻断性问题使用 `InfoBar.error`
  - 成功执行完成使用 `InfoBar.success`
  - 普通过程消息只进入任务日志，不弹全局提示
- 不再依赖 `statusBar()` 作为主要反馈通道；原有页面消息改为 `InfoBar + 任务日志` 双通道。

### 3. 设置页重构
- 设置页保留当前 `AppSettings` 和 `settings.json`，不迁移到 QFluentWidgets 的 `QConfig`。
- 页面改为标准 Fluent 设置中心样式，结构固定为：
  - 页头标题与说明
  - `ScrollArea`
  - 按一级分类生成 `SettingCardGroup`
- 控件映射规则固定如下：
  - 布尔值一律映射为 `SwitchSettingCard`
  - 有预定义选项的值一律映射为 `ComboBoxSettingCard`
  - 普通文本值映射为自定义 `LineEditSettingCard`
  - `api_key` 等敏感字段映射为密码输入型 `LineEditSettingCard`
- 保存策略固定为“即时保存”：
  - 值变化即调用现有 `save_settings()`
  - 成功保存不弹成功提示
  - 保存失败才弹 `InfoBar.error`
- 数据类型处理继续沿用 `AppSettings` 当前逻辑，不在本次整改中调整 JSON 结构或字段命名。
- 页面内不再混用 `QFormLayout`、`QHBoxLayout` 临时拼表单；所有设置项统一表现为卡片。
- 现有一级分组顺序保持不变，避免用户认知迁移成本。

### 4. 帮助页重构
- 帮助页保留当前 markdown 文档来源，不调整 `configs` 下文档文件格式。
- 页面结构固定为：
  - 标题区
  - 顶部 `SegmentedWidget`
  - 文档阅读卡片
- 页签固定为两项：`用户文档`、`开发文档`
- Markdown 继续用 `QTextBrowser` 渲染，外链保持系统浏览器打开。
- 默认打开 `用户文档`，从主导航进入帮助页时不自动切换文档，只有菜单操作或页签点击才切换。

## 接口与结构变更
- `MainWindow` 从旧的多部件组装类调整为 Fluent 应用壳，职责只保留：
  - 初始化主题
  - 注册导航页
  - 组装页面
  - 分发顶层通知
- 新增内部类型 `TaskDescriptor`，作为任务 UI 与任务注册的唯一元数据来源，替代硬编码元组。
- 保留 `BatchFilesBinding.result_signal(str task_name, str message)` 现有签名，不改业务线程接口。
- 保留 `AppSettings.changed_signal(str object_name, str attribute, object value)` 现有签名，不改配置传播接口。
- 新增设置页内部适配层，将 `AppSettings` 的 map 转换成设置卡片定义，避免页面直接处理路径和类型判断。
- 旧 `DockPage` 视为废弃组件，整改完成后不再被主窗口引用。

## 实施顺序
1. 接入 `PySide6-Fluent-Widgets==1.11.1`，完成启动导入、主题初始化、主窗口壳替换，先把旧页面原样挂进 Fluent 导航，验证可运行。
2. 引入统一主题模块和应用级 QSS，去掉现有页面里的局部 `setStyleSheet()` 和 `setGeometry()` 依赖，先解决整体视觉与间距不统一问题。
3. 重构任务中心，先落 `TaskDescriptor` 和新骨架，再把现有任务执行逻辑逐项挂接回来，保证功能不回退。
4. 重构设置页，用卡片体系替换现有动态表单，但保持现有配置加载和保存链路不动。
5. 重构帮助页，统一标题、分段切换和文档阅读体验。
6. 统一全局提示、错误态、空状态、加载态，最后清理旧导航、旧状态栏路径和无用 UI 代码。

## 测试计划
- 启动测试：
  - 安装新依赖后应用可正常启动
  - `FluentWindow` 可正常显示三页导航
  - 默认进入 `任务中心`
- 主题测试：
  - `settings.json` 中 `theme=Dark` 时以暗色主题启动
  - 切到 `Light` 后保存并重启，主题持久化正确
  - 自定义主题色在按钮、选中项、开关、强调状态上统一生效
- 任务中心测试：
  - 未选择目录时点击执行，出现错误提示且不崩溃
  - 选择目录并勾选子目录后，摘要数字正确
  - 切换不同任务时日志各自独立，不串台
  - 运行中的任务按钮禁用，其他任务不受影响
  - 任务完成、失败、重复点击运行中的任务，提示与日志行为符合预期
- 设置页测试：
  - 布尔、枚举、文本、密码字段均能正确显示与保存
  - 数字型枚举写回后类型不漂移
  - 保存失败时有错误提示，成功时静默
  - 切换分类滚动区域布局稳定，无控件错位
- 帮助页测试：
  - 用户文档与开发文档均可正确加载
  - 页签切换正常
  - 外部链接可打开系统浏览器
- 尺寸与可用性测试：
  - 在 `1280x720` 下无主要控件遮挡、重叠、跑版
  - 窗口缩放时导航、任务列表、日志区和设置卡片能正常伸缩
  - 长文本说明、长路径、长日志不破坏布局

## 默认假设
- 继续使用 GPLv3 分发，因此可合法引入该库的 GPLv3 许可版本。
- 以 `PySide6-Fluent-Widgets` 普通版为默认依赖，不使用 `full` 版和商业版。
- 现有模块处理逻辑、线程调度、参数注入和 JSON 配置格式均视为稳定，不纳入本次整改范围。
- 首版不追求 macOS 原生视觉，目标是统一、现代、模块化的跨平台工具界面。
- 首版不迁移到 `QConfig`，不新增高级搜索、拖拽排序、任务收藏、自定义首页等增强功能。
- 首版不保留旧 Dock 导航作为兼容模式，整改完成后直接切到 Fluent 导航体系。
